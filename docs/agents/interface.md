# Interface Layer Guide

대상: `src/interface/**`, 일부 `src/main.rs`

## 역할

사용자 입력(CLI)을 해석해 Application 유스케이스 호출로 연결한다.
출력 포맷은 최소화하고, 비즈니스 로직은 넣지 않는다.
Hexagonal 기준으로 `interface`는 인바운드 어댑터 계층이다.

## 포함 가능한 것

- CLI 인자 파싱(`clap`)
- 입력 검증(필수 인자 누락 등)
- 유스케이스 호출 분기

## 포함하면 안 되는 것

- 도메인 정책 계산
- HTTP/파일/프로세스 직접 실행
- 중복 방지/마커 판단 로직

## 설계 규칙

1. 인터페이스는 얇게 유지한다(Thin Interface).
2. 파싱 결과는 유스케이스 입력 타입으로 즉시 변환한다.
3. 에러 메시지는 사용자 친화적으로, 내부 상세는 로그로 분리한다.
4. 인바운드 포트/유스케이스 호출만 수행하고 아웃바운드 구현은 직접 호출하지 않는다.

## 리뷰 체크포인트

- CLI 계층이 usecase를 우회해 인프라를 직접 호출하는가?
- 인자 파싱과 비즈니스 결정을 섞고 있지 않은가?
